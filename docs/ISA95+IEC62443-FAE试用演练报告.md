# ISA95 + IEC62443 经验体 FAE 试用演练报告

## 1. 演练目标

本次演练以资深 FAE 的实际使用视角，对当前平台进行一次完整试用，目标包括：

- 用 `ISA95 + IEC62443` 经验体学习系统使用方式。
- 模拟一个真实工业项目，完成资料采集、判断依据、设计推演、结果落地、经验迭代的完整闭环。
- 观察系统是否能够同时支撑对外输出与对内沉淀。
- 评估当前经验体结构、结果页质量与经验回流机制是否具备继续扩展的基础。

## 2. 试用角色与场景

### 2.1 使用角色
本次试用角色设定为：工业网络规划方向资深 FAE。

关注重点为：

- 是否能按清晰步骤推进项目。
- 是否能把客户原始信息、判断依据与设计结论分开管理。
- 是否能把项目中的特殊判断沉淀为可复用经验。
- 是否能直接形成面向客户和面向内部的输出材料。

### 2.2 模拟项目
本次模拟项目名称：**离散制造产线改造与安全分区规划项目**。

项目背景如下：

- 某离散制造企业计划对现有产线网络进行升级。
- 现场同时存在 PLC、机器人、视觉检测、MES、工程师站与远程维护需求。
- 现有网络长期按“能通就行”的方式建设，边界不清、接入混乱、远程维护风险较高。
- 客户希望在不大幅影响生产的前提下，形成可实施的网络分层、分区、通道、安全接入与地址规划方案。

## 3. 如何学习这个系统

从 FAE 视角看，学习当前系统并不复杂，建议按以下顺序理解：

### 3.1 先理解平台不是单纯表单工具
这个平台不是只负责“填资料然后出文档”。

它更像一个工业网络规划经验表达框架，核心是把一次规划过程拆成五层：

- `资料采集`：先把现场情况问清楚。
- `判断依据`：明确在什么场景下按什么原则做决定。
- `设计推演`：基于前两层形成具体设计。
- `结果落地`：输出面向客户与内部的正式材料。
- `经验迭代`：把本项目中新增的判断与做法反哺回经验库。

### 3.2 再理解左侧菜单的意义
左侧菜单不是普通导航，而是一次规划项目的工作流。

对于 FAE 来说，它等价于：

- 我先采什么信息；
- 再看哪些规则；
- 再做哪些设计；
- 最后输出什么材料；
- 项目结束后哪些经验值得沉淀。

### 3.3 最后理解经验体的作用
`ISA95 + IEC62443` 不是一堆页面，而是一套已经组织好的规划经验。

它决定了：

- 这个项目要采哪些信息；
- 这个项目有哪些规则主题；
- 这个项目要做哪些推演页；
- 这个项目最终输出哪些文档；
- 这个项目如何沉淀经验。

因此，对 FAE 来说，学习系统的关键不是死记页面，而是理解一套经验体如何驱动整个规划闭环。

## 4. 按项目完整走一遍

## 4.1 资料采集阶段
在本次模拟项目中，FAE 首先会在资料采集层整理客户基础信息。

本次采集到的关键信息可概括为：

- 行业类型：离散制造。
- 建设范围：单厂区，多条生产线改造。
- 主要对象：PLC、机器人、工业交换机、视觉检测终端、MES 接口设备、工程师站。
- 网络现状：现场网络扁平，控制设备与管理终端混接，缺少清晰边界。
- 安全关注点：远程维护、工程师站越权接入、跨区访问缺少约束。
- 项目目标：在保证连续生产的前提下完成分层、分区、通道与接入优化。

从使用感受看，这一层的价值在于：

- 它逼着 FAE 先把事实收集完整；
- 它把“客户说了什么”和“我后面怎么设计”分开了；
- 后续很多页面可以直接复用这里的值，不需要重复输入。

## 4.2 判断依据阶段
进入判断依据层后，FAE 不再直接下设计结论，而是先看应该依据什么规则做判断。

在本次项目中，实际会重点使用到以下规则主题：

- `分层规则`：判断是否采用核心 / 汇聚 / 接入的多层结构。
- `区域规则`：判断控制区、监控区、管理区是否需要拆分。
- `通道规则`：判断跨区通信如何收敛。
- `边界互联规则`：判断 MES、远程运维与第三方系统如何受控接入。
- `技术选择规则`：判断产线侧采用何种工业交换与冗余方式。
- `安全规则`：判断远程维护、工程师站与安全域隔离策略。
- `地址规划规则`：判断 IP 分段与地址预留方式。

从 FAE 视角看，这一层最大的意义是：

- 避免“我觉得这样设计差不多”；
- 让设计变成“在什么场景下按什么规则做决定”；
- 让团队成员之间的方案差异变得可讨论、可复盘。

## 4.3 设计推演阶段
本阶段是系统真正体现专业价值的地方。

在本次项目中，FAE 会基于前序信息与规则，逐项形成设计结论。

### 4.3.1 分层设计
由于项目为单厂区但对象数量较多，且多个产线单元相对独立，因此采用三层思路更合理：

- 核心层：厂区中心机房，承载管理互联、MES 接入与统一汇聚。
- 汇聚层：按车间或产线区域设置汇聚节点。
- 接入层：连接 PLC、机器人、视觉终端与工程师站。

### 4.3.2 区域划分
结合 IEC62443 的安全视角，对现场对象进行分区：

- 生产控制区：PLC、机器人控制单元。
- 过程监控区：HMI、SCADA、产线监控终端。
- 工程维护区：工程师站、调试终端。
- 信息交互边界区：MES 接口、历史数据库接口、受控数据交换节点。
- 远程接入边界区：远程维护跳板、审计与认证接入点。

### 4.3.3 通道与边界设计
为避免跨区随意互通，通道设计原则为：

- 控制区与监控区按业务白名单开放必要协议。
- 工程维护区不得直接长期驻留在控制网络。
- MES 仅通过边界节点获取所需数据。
- 远程维护必须经过受控接入路径，并保留审计能力。

### 4.3.4 技术选择
结合离散制造现场的实时性与稳定性要求，本项目技术选择倾向为：

- 产线侧优先采用工业交换机。
- 对关键链路考虑环网或双链路冗余。
- 对高实时对象优先考虑低时延、明确广播域边界的接入方式。
- 对办公与控制混接风险较高的区域，优先拆分接入网络。

### 4.3.5 地址规划
地址规划按区域与用途切分，避免后续扩展时地址混乱：

- 控制对象地址独立规划。
- 监控终端地址独立规划。
- 工程维护终端地址独立规划。
- 边界与远程接入地址独立规划。
- 每类地址段保留扩展余量。

从使用感受看，设计推演层已经基本符合 FAE 的真实工作方式，因为它不是简单填空，而是在逐步收敛设计。

## 5. 结果落地演练

基于上述模拟信息，系统最终应能形成面向客户和面向内部的输出。

以下内容为本次演练对应的一版成稿示意。

## 5.1 面向客户

### 5.1.1 现场概述
本项目面向某离散制造企业产线升级场景开展工业网络规划，建设范围为单厂区多产线改造。现场包含 PLC、机器人、视觉检测终端、MES 接口设备、工程师站等多类对象，现网存在控制对象与管理终端混接、边界不清、远程维护路径不受控等问题。

本次规划的目标，是在不显著影响生产连续性的前提下，对网络进行结构梳理与安全整治，形成可实施的分层、分区、通道、接入与地址规划方案，为后续建设、改造和验收提供统一依据。

### 5.1.2 设计说明
本次设计采用业务结构与工业安全结合的方式进行推演。整体网络按核心、汇聚、接入三个层次组织，厂区中心机房承担统一汇聚与对外互联功能，车间或产线按业务边界设置汇聚节点，现场设备侧采用清晰的接入组织方式。

在安全组织上，规划将现场对象划分为生产控制区、过程监控区、工程维护区、信息交互边界区和远程接入边界区，并针对不同区域定义访问边界和互联方式。MES、远程维护与工程维护等高风险访问路径均通过受控边界接入，避免跨区无约束互通。

### 5.1.3 网络拓扑
建议拓扑采用“厂区核心 + 车间汇聚 + 产线接入”的总体结构。控制对象就近接入本产线交换节点，再汇聚至车间级节点，由车间级节点统一上联至核心区域。MES、历史数据库及远程维护能力通过独立边界接入，不直接深入控制侧网络。

该拓扑方案兼顾了离散制造现场的可实施性、维护便利性与后续扩展能力，便于在不一次性大改现场的情况下分阶段实施。

### 5.1.4 IP 规划
建议按区域与用途进行独立地址规划。控制对象、监控对象、工程维护终端、边界互联节点与远程接入节点分别使用不同地址段，并为后续扩线、设备增加和安全隔离调整保留地址余量。

该规划方式有利于提升后续运维识别效率，也便于策略编排、边界控制和故障定位。

## 5.2 面向内部

### 5.2.1 规划复盘
本项目的关键设计收敛点主要有三项。

第一，现场原有网络偏扁平，如果仍沿用“大二层混接”方式，后续在安全隔离、故障定位和扩线时都会持续放大风险，因此必须先完成结构分层。

第二，控制对象、工程维护对象和信息交互对象的风险性质不同，若不提前进行分区与边界定义，则后续所有安全措施都只能停留在补丁式加固，难以形成稳定结构。

第三，客户提出希望尽量少停线施工，因此本项目方案优先考虑分阶段落地，先完成网络边界和汇聚结构调整，再逐步推进设备侧梳理与接入收敛。

### 5.2.2 案例提炼
本案例适用于离散制造产线改造场景，尤其适用于以下条件：

- 现场对象种类多，控制与管理混接明显；
- 客户同时存在 MES 接入与远程维护诉求；
- 现网历史包袱较重，难以一次性整体替换；
- 需要兼顾安全改造与实施可行性。

本案例可沉淀的经验包括：

- 单厂区多产线场景下，优先按“核心 + 汇聚 + 接入”组织网络；
- 涉及 MES、远程维护与工程师站时，应优先从边界路径治理入手；
- 工程维护路径不应默认等同于控制路径；
- 地址规划应在设计初期同步完成，否则后续边界治理成本会上升。

## 6. 经验迭代评估

从本次演练看，经验迭代层是这个系统区别于普通规划工具的关键。

在本项目中，至少可以沉淀出以下几类回流内容：

### 6.1 特殊判断
- 当客户要求“尽量不停线改造”时，设计优先级应从“最理想结构”调整为“可分阶段落地结构”。
- 当离散制造现场同时存在机器人、视觉与 MES 接口时，边界治理优先级通常高于纯地址规划优先级。

### 6.2 经验候选
- 离散制造多产线改造项目中，边界治理往往是最容易形成客户感知价值的切入点。
- 若现网长期混接，工程维护区应单独显式化，而不应隐含在普通接入中。

### 6.3 迭代总结
本次项目说明：`ISA95 + IEC62443` 这套经验体适合处理“业务结构复杂 + 安全边界混乱 + 需要兼顾实施可行性”的工业项目，但后续还可继续增强以下内容：

- 不同行业场景下的默认规则组合；
- 更多典型对象的分区模板；
- 更明确的推演页与规则主题引用关系；
- 更成熟的结果页正式文风模板。

## 7. 作为资深 FAE 的使用感受

### 7.1 优点
当前系统已经具备明显优势：

- 左侧结构清晰，确实能把一次规划过程组织起来。
- 资料、规则、推演、结果、迭代五层已经形成闭环。
- 多记录模式适合真实项目，不会被单条表单限制。
- 经验体可以复用，适合后续扩展更多行业和对象。
- 结果页已经开始具备正式交付文风，而不是机械字段拼接。

### 7.2 当前不足
从真实使用角度，当前还存在几个明显短板：

- `ISA95 + IEC62443` 的内容深度还没有完全覆盖旧系统的全部成熟细节。
- 判断依据层虽然已有较完整主题树，但不同主题下的规则厚度还不均衡。
- 设计推演页与规则主题之间，当前还是“可引用”，还没有更强的联动提示。
- 结果落地页已经可用，但距离真正高保真、可直接交付客户，还有进一步润色空间。
- 经验迭代层现在已经有结构，但回写动作仍以人工整理为主，还没有半自动升级能力。

## 8. 对平台下一步的建议

如果按产品化和长期可用角度看，下一步最值得做的事情有四项：

### 8.1 先补厚标杆经验体
优先把 `ISA95 + IEC62443` 继续做厚，尤其是：

- 判断依据层规则内容；
- 设计推演层关键页细节；
- 结果落地层正式化模板；
- 经验迭代层回写建议模板。

### 8.2 强化规则到推演的联动
当用户进入某个推演页时，系统应更明确告诉他：

- 当前页主要参考哪些规则主题；
- 当前常见情形有哪些默认做法；
- 当前若偏离常见做法，建议记录到经验迭代层。

### 8.3 让经验迭代更像真实闭环
后续可以把“特殊判断”“经验候选”进一步结构化，例如：

- 来源项目；
- 适用场景；
- 触发条件；
- 推荐动作；
- 是否建议升级为正式规则；
- 影响到哪些推演页或结果页。

### 8.4 继续提升结果页正式度
如果平台要真正成为团队协作工具，结果页必须继续提升，做到：

- 面向客户的文风更正式、更稳定；
- 面向内部的复盘更像方法总结，而不是信息摘录；
- 同一经验体下的输出风格统一。

## 9. 演练结论

从资深 FAE 视角评价，当前系统已经不是简单原型，而是一个已经完成主框架收敛的工业网络规划经验表达平台。

它最重要的价值，不在于“有很多页面”，而在于已经把一次工业网络规划过程稳定拆成：

- 资料采集
- 判断依据
- 设计推演
- 结果落地
- 经验迭代

这五层闭环已经足以支撑后续长期扩展。以后无论增加新的经验体、行业场景、系统对象还是技术方向，基本都不会跳出这套框架。

从当前阶段判断，平台下一步不再是继续大改总架构，而是围绕标杆经验体做内容补厚、推演联动增强和结果页正式化提升。
