# 方法论定义格式草案

## 目标
方法论文件应成为系统真正的内容真源，用来定义：
- 有哪些菜单
- 每个菜单对应什么页面
- 页面由哪些组件组成
- 字段默认值、选项和依赖关系是什么
- 页面影响哪些输出

## 最小定义层次
- 方法论元信息
- 导航树
- 页面定义
- 字段定义
- 输出映射

## 示例骨架
```json
{
  "meta": {
    "key": "industrial-network-core",
    "title": "工业网络规划系统",
    "subtitle": "方法论驱动的内容配置框架"
  },
  "navigation": [
    {
      "key": "project",
      "title": "01. 项目目标",
      "children": [
        {
          "key": "project-basic",
          "title": "01.1 基础信息",
          "page": "standard-record"
        }
      ]
    }
  ],
  "pages": {
    "project-basic": {
      "type": "standard-record",
      "entity": "projectBasic",
      "summaryColumns": ["name", "customer", "industry"],
      "fields": ["name", "customer", "industry"]
    }
  },
  "fields": {
    "projectBasic.name": {
      "label": "项目名称",
      "type": "text",
      "defaultValue": "工业网络规划项目"
    }
  }
}
```
