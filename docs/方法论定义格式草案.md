# 方法论配置格式草案

## 当前状态说明
本文件保留为早期草案说明。
当前项目建议以以下文档为主：
- `docs/methodology-schema-v1.md`

## 方法论配置的职责
方法论配置文件应成为平台中的内容真源，用来定义：
- 系统元信息
- 左侧导航树
- 页面定义
- 页面内部结构
- 字段定义
- 依赖关系
- 输出映射
- 方法论自带案例

## 当前最小方向
一个方法论配置当前建议至少包含：
- `meta`
- `systemPages`
- `navigation`
- `pages`
- `fields`
- `outputs`
- `cases`

## 当前结论
- 平台本身不应因不同方法论配置而大改
- 应通过替换不同方法论配置来切换内容
- 当前开发阶段允许先用 `.js`
- schema 稳定后可逐步收敛为 `.json`
- 不同方法论配置应体现不同菜单结构和内容重点
- 方法论配置未来将成为平台中知识沉淀与协作的主要载体
