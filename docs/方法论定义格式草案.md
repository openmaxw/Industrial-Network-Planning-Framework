# 方法论定义格式草案

## 当前状态说明
本文件保留为早期草案说明。
当前项目已经进入更明确的 schema 阶段，建议以以下文档为主：
- `docs/methodology-schema-v1.md`

## 方法论的职责
方法论文件应成为系统真正的内容真源，用来定义：
- 系统元信息
- 左侧导航树
- 页面定义
- 页面内部结构
- 字段定义
- 依赖关系
- 输出映射
- 方法论自带案例（后续）

## 当前最小方向
一个 methodology 当前建议至少包含：
- `meta`
- `systemPages`
- `navigation`
- `pages`
- `fields`
- `outputs`

## 当前结论
- 框架本身不应因不同方法论而大改
- 应通过替换不同 methodology 来切换系统内容
- 当前开发阶段允许先用 `.js`
- schema 稳定后可逐步收敛为 `.json`
