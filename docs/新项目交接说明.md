## 2. 历史背景
旧项目是一个已经较完整的工业网络规划业务系统，可视为当前可用的业务成品基线。新项目不再在旧系统上继续硬改，而是转向构建一个方法论驱动的内容配置框架。

旧项目参考位置：
- 本地目录：`/home/max/ISA95_IEC62443_Planner`
- GitHub 仓库：`git@github.com:openmaxw/ISA95_IEC62443_Planner.git`

---

## 3. 新项目定位
新项目目标不是继续做固定内容系统，而是做一个稳定框架，使以下内容由 methodology 驱动：
- 左侧导航树
- 页面结构
- 页面内部布局
- 字段与默认值
- 依赖关系
- 输出映射
- 案例数据

也就是说：
- 不变的是框架
- 可变的是 methodology

---

## 4. 技术路线
已确定：
- 采用 `React + Vite`
- 在独立目录中开发
- 采用“框架固定、methodology 可变”的方向
- 系统先进入首页，再选择具体方法论

## 6. 当前架构理解
当前项目可按四层理解：
- `App`：应用壳层
- `methodology`：内容定义层
- `engine`：定义解析层
- `store`：运行状态层

同时还要加一个系统层理解：
- 首页与方法论选择属于框架层，不属于某个具体方法论页面

---

## 7. 当前方法论状态
当前已存在三套最小方法论入口：
- `ISA95`
- `IEC62443`
- `ISA95 + IEC62443`

它们已经不是同构占位，而是具备不同的：
- 菜单结构
- 页面内容
- 字段重点
- 案例数据

---

## 8. 当前最重要的开发原则
- 不要再把新框架做成固定业务系统
- 不要优先复制旧页面 HTML
- 要优先定义 methodology 如何描述系统
- 控制权优先留在 methodology，而不是实现细节里
- 首页和方法论页要明确分层
