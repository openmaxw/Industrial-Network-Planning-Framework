# 新项目交接说明

## 1. 本文件用途
本文件用于让新的模型或开发者在不依赖历史对话的前提下，快速理解：
- 这个项目当前的定位是什么
- 当前已经做到哪一步
- 接下来应如何继续推进

建议与以下文档配合阅读：
- `docs/项目定位与架构说明.md`
- `docs/experience-schema-v1.md`

---

## 2. 当前定位变化
项目最初的目标是“经验驱动的内容配置框架”。
当前随着首页、角度入口、多经验配置、多案例与导入导出能力逐步建立，定位已经进一步提升为：

**工业网络规划知识表达框架**

这意味着平台不只是为了运行一套固定内容，而是为了：
- 沉淀经验配置
- 支持多视角组织
- 支持多人协作与经验复用
- 为经验收敛提供载体

---

## 3. 当前已完成的骨架
当前项目已经完成以下核心能力：
- 平台首页
- 平台介绍区
- 顶部角度导航
- 首页按角度切换区块
- 内置经验配置选择
- 外部经验配置 JSON 导入
- 当前经验配置 JSON 导出
- 当前运行数据 JSON 导出
- 经验驱动左侧菜单
- 统一记录页模板
- 输出页模板
- 经验配置基础校验
- 多套经验配置入口

---

## 4. 当前平台层与内容层分工
当前项目应分成两层理解：

### 平台层
负责：
- 首页
- 平台介绍
- 角度分类入口
- 经验配置导入导出
- 经验配置组织与浏览

### 内容层
负责：
- 某个具体经验配置内部的菜单
- 页面
- 字段
- 案例
- 输出

平台层不等于经验层，这一点是当前项目最重要的结构共识之一。

---

## 5. 当前架构理解
当前项目仍然可以按四层理解：
- `App`：平台壳层
- `methodology`：经验配置层
- `engine`：定义解析层
- `store`：运行状态层

但相比早期阶段，现在更应该先从“平台层 / 内容层”视角理解整个系统。

---

## 6. 当前经验配置状态
当前已存在三套最小经验配置入口：
- `ISA95`
- `IEC62443`
- `ISA95 + IEC62443`

它们已具备不同的：
- 菜单结构
- 页面重点
- 案例内容

综合方法论中，已经开始做旧系统“项目目标”页的迁移样板。

---

## 7. 当前最重要的开发原则
- 平台名固定，不随当前经验配置改变
- 首页默认显示平台介绍区
- 顶部角度入口用于平台级浏览
- 左侧导航用于当前经验内部导航
- 经验配置应逐步成为经验沉淀与协作的主要载体

---

## 8. 下一步建议顺序
1. 继续高保真迁移旧系统关键页面
2. 将综合经验配置逐步做实
3. 补更多场景角度、系统对象角度、技术方向角度的经验配置入口
4. 继续加强经验配置校验
5. 后续再考虑方法论对比、差异分析与协作增强
