# 经验配置格式规范 V1（草案）

## 目标
本文件用于定义当前框架中 `experience` 的最小稳定结构。
它描述的是“系统内容如何被定义”，而不是“界面如何实现”。

---

## 1. 顶层结构
一个 `experience` 建议包含以下顶层字段：

```json
{
  "meta": {},
  "systemPages": {},
  "navigation": [],
  "pages": {},
  "fields": {},
  "outputs": {},
  "cases": []
}
```

说明：
- `meta`：经验元信息
- `systemPages`：系统级页面定义
- `navigation`：左侧导航树
- `pages`：页面定义
- `fields`：字段定义
- `outputs`：输出定义
- `cases`：案例数据

---

## 2. 元信息（meta）
用于定义经验体本身的基本信息。

建议字段：
- `key`
- `title`
- `subtitle`
- `version`
- `description`

---

## 3. 导航树（navigation）
用于定义左侧菜单树。

当前平台固定骨架已采用五层：
- 资料采集
- 判断依据
- 设计推演
- 结果落地
- 经验迭代

经验配置中的 `navigation` 应优先围绕这五层展开。

示例：
```json
[
  {
    "key": "how-to-ask",
    "title": "资料采集",
    "children": []
  },
  {
    "key": "how-to-iterate",
    "title": "经验迭代",
    "children": []
  }
]
```

---

## 4. 页面定义（pages）
用于定义每个页面是什么类型，以及页面包含什么。

当前建议支持的页面类型：
- `standard-record`
- `record-collection`
- `group`
- `output`
- `tool`
- `home`

说明：
- `standard-record`：单页字段编辑页
- `record-collection`：可录入多条记录的集合页
- `group`：目录页/分组页，本身不直接编辑数据
- `output`：输出展示页
- `tool`：工具页
- `home`：首页或系统级说明页

---

## 5. 字段定义（fields）
用于定义字段元信息。

建议字段：
- `label`
- `type`
- `defaultValue`
- `options`
- `optionsSource`
- `visibleWhen`
- `required`
- `description`

当前建议支持的字段类型：
- `text`
- `textarea`
- `select`
- 后续可扩展 `number` / `checkbox` / `radio` / `table`

---

## 6. 字段依赖
### 6.1 显示依赖
用于控制字段是否显示。

### 6.2 数据依赖
用于让某字段的选项来自其他字段或记录。

当前系统已支持典型场景：
- 从普通字段多行内容生成选项
- 从记录页某列提取选项
- 对重复值去重

---

## 7. 输出定义（outputs）
用于定义输出页各 section 的内容来源。

当前系统支持：
- `key-value-summary`
- `record-summary`
- `text-summary`

并且已经支持：
- 从单页字段拼接输出
- 从单个记录页汇总输出
- 从多个记录页自动汇总输出

这使得经验迭代层可以自动汇总设计推演页中的回流标记。

---

## 8. 经验迭代相关字段
当前系统已引入经验迭代机制，建议经验配置中支持以下字段：
- `derive.iterationFlag`
- `derive.iterationNote`
- `derive.candidateSuggestion`
- `iteration.*`
- `candidate.*`

它们用于表达：
- 推演页是否形成经验回流项
- 推演页中的特殊说明
- 建议沉淀到经验库的内容
- 特殊判断记录
- 经验候选记录

---

## 9. 当前落地建议
当前阶段建议：
- 先允许 `experience` 使用 `.js`
- 当格式规范稳定后，再收敛为 `.json`
- 所有灵活内容优先进 `experience`
- 所有通用逻辑优先进 `engine`
- 所有运行时数据优先进 `store`

---

## 10. 当前结论
这个项目未来最重要的可配置核心，就是 `experience`。
框架本身不应该因为不同经验体而大改，应该通过替换不同 `experience` 来切换系统内容。

而在当前阶段，经验配置已经不只是定义页面和字段，
也开始承担“经验发现、经验回流与经验迭代”的结构化表达职责。
