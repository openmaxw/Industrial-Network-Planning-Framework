# 工业网络规划系统

## 副标题
方法论驱动的内容配置框架

## 项目定位
本项目不是旧业务系统的继续堆叠，而是一个独立的前端框架项目。它的目标是提供一套稳定框架，使不同方法论、行业场景和页面结构能够通过内容定义文件加载出来，而不是把业务细节直接写死在页面与脚本中。

## 核心原则
- 框架稳定，方法论可变。
- 菜单树、页面、字段、默认值、依赖关系、输出映射由内容定义驱动。
- 系统先启动首页，再由用户选择方法论进入业务内容。
- 左侧导航树是方法论激活后的主导航。
- 标准页优先统一，特例页最小化。

## 当前已落地的架构分层
- `src/app/`：应用壳层，负责整体布局与模块装配。
- `src/components/`：通用页面组件，当前已包含首页、标准页、记录页、输出页。
- `src/methodologies/`：方法论定义文件，当前已拆出 `ISA95`、`IEC62443` 和 `ISA95 + IEC62443` 三套入口。
- `src/engine/`：方法论解析辅助逻辑。
- `src/store/`：运行时状态管理，负责当前方法论、页面状态和表单数据。
- `src/themes/`：视觉样式。
- `docs/`：架构与方法论文档。

## 当前已实现的最小链路
1. 系统首页骨架
2. 首页选择方法论
3. 方法论驱动的左侧多级菜单
4. 标准记录页渲染
5. 记录集合页渲染
6. 字段默认值、显示依赖、动态选项来源
7. 输出页汇总展示
8. 方法论自带案例加载
9. 当前数据导出
10. `methodology / engine / store / app` 的初步分层

## 当前界面结构
- 顶部：系统区
- 左侧：导航菜单区
- 右侧：内容区
- 系统启动默认进入首页
- 未选择方法论前，不显示业务菜单

## 当前方法论状态
当前已存在三套最小方法论入口：
- `ISA95`：偏企业层级、业务流程、对象分配
- `IEC62443`：偏安全区域、通道、威胁、风险
- `ISA95 + IEC62443`：偏综合规划
