# 工业网络规划系统

## 副标题
方法论驱动的内容配置框架

## 项目定位
本项目不是现有业务系统的继续堆叠，而是下一代独立框架项目。它的目标是提供一套稳定的前端框架，使不同方法论、行业场景和页面结构能够通过内容定义文件加载出来，而不是把所有业务细节直接写死在页面与脚本中。

## 核心原则
- 框架稳定，方法论可变。
- 菜单树、页面、组件、默认值、依赖关系、输出映射由内容定义文件驱动。
- 左侧导航树是系统唯一主导航，不再保留页面内 tab 作为主要结构。
- 标准记录页优先统一，特例页最小化。
- 旧项目作为业务基线，新项目作为框架基线，两者分离维护。

## 目标结构
- `src/app/`：应用入口与壳层。
- `src/engine/`：根据方法论定义生成导航、页面与交互的引擎。
- `src/components/`：通用 UI 组件。
- `src/methodologies/`：方法论定义文件。
- `src/outputs/`：输出生成与映射逻辑。
- `src/store/`：状态管理。
- `src/themes/`：视觉样式。
- `src/utils/`：通用工具。
- `docs/`：框架设计与方法论规范。

## 第一阶段目标
1. 定义方法论文件的最小格式。
2. 实现左侧多级菜单渲染。
3. 实现标准记录页渲染器。
4. 实现字段默认值、枚举与依赖关系读取。
5. 以一组最小样例页面验证引擎链路。
