# 项目定位与架构说明

## 项目定位
本项目是一个面向工业网络规划场景的**经验沉淀、表达、复用、协作与迭代框架**。

它当前已经明确采用：
- 资料采集
- 判断依据
- 设计推演
- 结果落地
- 经验迭代

这五层组成一个显式闭环，使系统不仅能完成规划过程，也能把项目中新产生的经验持续回流到经验库。

它的目标是：
- 让不同经验可以被结构化表达
- 让不同项目场景下的做法可以被沉淀与复用
- 让团队成员可以围绕经验进行共享、讨论与持续优化
- 让经验不仅能驱动采集、推演与输出，还能从项目中持续回流迭代

## 核心原则
- 平台固定，经验可变。
- 首页属于平台层，经验页面属于内容层。
- 菜单、页面、字段、依赖关系、案例和输出结构由经验配置驱动。
- 平台支持经验加载、导入、导出和持续演进。
- 左侧导航用于当前经验内部导航，顶部入口用于平台级切换。
- 系统已从四层结构升级为五层闭环结构。

## 当前已落地的架构分层
- `src/app/`：平台壳层，负责整体布局、首页入口与模块装配。
- `src/components/`：页面组件，当前包含首页、记录页、输出页等。
- `src/methodologies/`：经验配置库，当前承载多套经验配置。
- `src/engine/`：经验配置解析与校验逻辑。
- `src/store/`：运行时状态管理。
- `src/themes/`：平台样式。
- `docs/`：平台与经验文档。

## 当前五层骨架
当前平台已固定为以下五层：
- 资料采集
- 判断依据
- 设计推演
- 结果落地
- 经验迭代

其中：
- 前四层负责完成项目规划流程
- 第五层负责经验回流与经验升级

## 当前经验配置状态
当前已存在 5 套可用经验配置：
- `ISA95`
- `IEC62443`
- `ISA95 + IEC62443`
- `储能项目`
- `MES`

其中 `ISA95 + IEC62443` 作为标杆经验体，已完整覆盖：
- 资料采集
- 判断依据
- 设计推演
- 结果落地
- 经验迭代

## 当前规则层与推演层关系
当前系统已明确采用：
- 规则层按主题分类
- 推演层按决策过程展开

也就是说：
- 判断依据层不追求和推演页机械一一对应
- 但要求每个关键推演决策点都有稳定的规则主题归属

在标杆经验体 `ISA95 + IEC62443` 中，已经开始建立：
- 拓扑规则 ↔ 网络拓扑
- 分层规则 ↔ 分层设计
- 区域规则 ↔ 区域划分
- 通道规则 ↔ 通道设计
- 边界互联规则 ↔ 边界互联
- 技术选择规则 ↔ 技术选择
- 地址规划规则 ↔ 地址分段
- 性能规则 ↔ 性能设计 / 时延分析 / 性能评估
- 稳定性规则 ↔ 稳定性设计
- 冗余规则 ↔ 冗余设计
- 接入规则 ↔ 对象接入
- 选型规则 ↔ 设备选型
- 部署规则 ↔ 部署设计
- 安全规则 ↔ 区域 / 通道 / 边界 / 远程接入类决策
- 场景规则 ↔ 设计原则与跨页综合判断

## 当前经验迭代机制
当前系统中的经验迭代层已具备以下结构：
- 特殊判断
- 经验候选
- 迭代总结

并且设计推演页已开始显式支持：
- 是否形成经验迭代项
- 推演中的特殊说明
- 建议沉淀内容

这使得系统具备以下闭环：
- 在推演中发现新经验
- 在迭代层中汇总与整理
- 再回写到判断依据、设计推演、案例提炼或经验元数据

并且当前标杆经验体已经开始显式支持：
- 推演页记录引用规则主题
- 推演页记录是否形成经验迭代项
- 经验迭代页自动汇总推演页中的规则引用与回流标记
