# 工业网络规划系统

## 副标题
方法论驱动的内容配置框架

## 项目定位
本项目不是旧业务系统的继续堆叠，而是一个独立的前端框架项目。它的目标是提供一套稳定框架，使不同方法论、行业场景和页面结构能够通过内容定义文件加载出来，而不是把业务细节直接写死在页面与脚本中。

## 核心原则
- 框架稳定，方法论可变。
- 菜单树、页面、字段、默认值、依赖关系、输出映射由内容定义驱动。
- 左侧导航树是系统唯一主导航，不再保留页面内 tab 作为主要结构。
- 标准页优先统一，特例页最小化。
- 旧项目作为业务基线，新项目作为框架基线，两者分离维护。

## 当前已落地的架构分层
- `src/app/`：应用壳层，负责整体布局与模块装配。
- `src/components/`：通用页面组件，当前已包含首页、标准页、记录页、输出页。
- `src/methodologies/`：方法论定义文件，当前已有示例 `demoMethodology.js`。
- `src/engine/`：方法论解析辅助逻辑。
- `src/store/`：运行时状态管理，当前已抽出最小 store。
- `src/themes/`：视觉样式。
- `docs/`：架构与方法论文档。

## 当前已实现的最小链路
1. 系统首页骨架
2. 左侧多级菜单渲染与展开收起
3. 标准记录页渲染
4. 记录集合页渲染
5. 字段默认值、显示依赖、动态选项来源
6. 输出页汇总展示
7. `methodology / engine / store / app` 的初步分层

## 当前界面结构
- 顶部：系统区
- 左侧：导航菜单区
- 右侧：内容区
- 系统启动默认进入首页
